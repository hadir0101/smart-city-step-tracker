SQLite Database

Migrated from parsing XML on every request to a persistent SQLite database. 720 hourly records for April 2026 seeded automatically. UNIQUE constraint prevents duplicates on re-ingest.

Two-Pipeline Ingestion Layer

Modular ingestion/ package with two independent sections — CSV file export and live Phyphox HTTP API. Both feed the same detection algorithm. Clean REST endpoints for each.

Step Detection Algorithm

Custom scipy-based peak detection on raw accelerometer data. Butterworth low-pass filter removes noise, find_peaks() identifies each footstrike. Works with any phone orientation.

Signal Processing
Step Detection Algorithm
Raw accelerometer → filtered signal → peak detection → step count
📡
3-Axis Magnitude

√(ax² + ay² + az²) — removes phone orientation dependency

↓
〰️
Butterworth Low-Pass Filter

Cutoff 3 Hz — walking is 1–2 Hz, noise is higher frequency

↓
▲
Peak Detection

scipy find_peaks — min gap 0.3s, threshold = mean + 0.4σ

↓
✅
Result

Step count · Cadence (steps/min) · Duration · Sampling rate

# 1. Compute magnitude
df['magnitude'] = np.sqrt( df['ax']**2 + df['ay']**2 + df['az']**2)

# 2. Low-pass filter (3 Hz cutoff)
b, a = butter(4, 3.0 / (0.5 * fs), btype='low')
df['filtered'] = filtfilt(b, a, df['magnitude'])

# 3. Detect peaks (each peak = 1 step)
threshold = df['filtered'].mean() + df['filtered'].std() * 0.4
peaks, _ = find_peaks(
df['filtered'],
height=threshold,
distance=int(fs * 0.3) # min 0.3s between steps
)

# 4. Results
steps = len(peaks)
cadence = steps / (duration / 60.0)

Algorithm Validation
Cadence-Based Verdict System
Validates detection accuracy using physiological walking norms
GOOD

80–140 steps/min

Normal walking pace. Detection is accurate and reliable.

ACCEPTABLE

60–80 or 140–180 steps/min

Slow stroll or brisk walk. Slightly outside typical range but plausible.

NEEDS TUNING

< 60 or > 180 steps/min

Outside physiological range. Adjust detection threshold or filter cutoff.
